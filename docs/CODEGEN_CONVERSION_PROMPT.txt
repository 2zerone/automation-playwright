# Codegen 테스트 코드 변환 요청 프롬프트

## 사용 방법
1. Playwright Codegen을 통해 원하는 테스트를 수행하고 생성된 코드를 복사
2. classes 폴더에 임의의 파일명으로 새 파일을 생성하고 코드를 붙여넣기
3. 아래 프롬프트를 사용하여 변환 요청

---

## 프롬프트 템플릿

---

## 상세 프롬프트 (더 구체적인 요청)

```
다음 codegen 테스트 코드를 classes 폴더의 Manager 클래스 패턴으로 변환해주세요.

**변환 대상 파일**: TicketCreateManager.js


**변환 요구사항**:

1. **클래스 구조**:
   - BaseManager를 상속받는 ES6 클래스로 변환
   - constructor(utils) 형태의 생성자 사용
   - module.exports = ClassName 형태로 내보내기

2. **메서드 구조**:
   - 비동기 메서드들을 의미있는 이름으로 그룹화
   - 각 메서드는 try-catch 블록으로 감싸기
   - console.log로 시작/완료/실패 메시지 추가

3. **에러 처리**:
   - 모든 메서드에 try-catch 블록 추가
   - 에러 발생 시 적절한 에러 메시지와 함께 throw
   - 실패 시 스크린샷 캡처 (captureFailureScreenshot)

4. **스크린샷 캡처**:
   - 성공 시: captureSuccessScreenshot(stepName)
   - 실패 시: captureFailureScreenshot(stepName)

5. **코딩 스타일**:
   - 기존 classes 폴더의 다른 파일들과 동일한 스타일 유지
   - 적절한 주석 추가
   - 일관된 들여쓰기와 포맷팅

6. **동적 값 처리**:
   - 업무코드, 저장소명, 프로젝트명 등 랜덤값은 하드코딩하지 말고 config에서 가져오기
   - 테스트 수행 중 생성된 값들은 적절한 변수나 config 객체를 통해 참조
   - 예시: `config.project.code`, `config.repository.name`, `config.user.id` 등

**참고 파일들**:
- lib/classes/BaseManager.js
- lib/classes/UserManager.js
- lib/classes/LoginManager.js
- lib/classes/TromboneUtils.js

위 파일들의 구조와 패턴을 참고하여 동일한 형태로 변환해주세요.
```

## 주의사항

1. **파일명 규칙**: Manager로 끝나는 명명 규칙 권장
2. **메서드명**: 의미있는 이름으로 그룹화 (예: executeTest, performLogin 등)
3. **에러 메시지**: 한국어로 명확한 에러 메시지 작성
4. **스크린샷**: 각 단계별로 적절한 stepName 사용
5. **로깅**: 이모지와 함께 진행 상황을 명확히 표시

2단계 → 































**동적 값 처리**: 
   - 업무코드, 저장소명, 프로젝트명 등은 config 객체에서 가져오기
   - 하드코딩된 값들을 config 기반으로 변경
   - 테스트 중 생성되는 ID나 코드는 적절한 변수로 처리

## 동적 값 처리 예시

### ❌ 잘못된 예시 (하드코딩)
```javascript
await this.utils.fillInput('input[placeholder="업무코드 선택"]', 'TROMBONE-PROJECT-001');
await this.utils.fillInput('input[placeholder="저장소명"]', 'trombone-repo-12345');
```

### ✅ 올바른 예시 (config 기반)
```javascript
// config에서 동적 값 가져오기
const taskCodeText = `${config.project.name}(${config.project.code})`;
const repositoryName = config.repository.name;

await this.utils.fillInput('input[placeholder="업무코드 선택"]', taskCodeText);
await this.utils.fillInput('input[placeholder="저장소명"]', repositoryName);
```

### 동적 값이 필요한 경우들
- 업무코드: `config.project.code`
- 프로젝트명: `config.project.name`
- 저장소명: `config.repository.name`
- 사용자 ID: `config.user.id`
- 사용자명: `config.user.name`
- 이메일: `config.user.email`
- 테스트 중 생성되는 ID나 코드: 적절한 변수로 처리
