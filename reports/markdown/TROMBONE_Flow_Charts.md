# TROMBONE UI 테스트 자동화 시스템 플로우 차트

## 1. 전체 시스템 플로우

```mermaid
flowchart TD
    A[사용자 GUI 실행] --> B[시나리오 선택]
    B --> C[설정 파일 로드]
    C --> D[Playwright 브라우저 시작]
    D --> E[로그인 수행]
    E --> F{로그인 성공?}
    F -->|실패| G[에러 처리 및 재시도]
    G --> E
    F -->|성공| H[시나리오별 테스트 실행]
    
    H --> I[업무코드 등록]
    I --> J[툴체인 등록]
    J --> K[저장소 등록]
    K --> L[사용자 업무코드 부여]
    L --> M[SonarQube 등록]
    M --> N[JUnit 등록]
    N --> O[파이프라인 등록]
    O --> P[워크플로우 컴포넌트 등록]
    
    P --> Q{테스트 성공?}
    Q -->|실패| R[스크린샷 캡처]
    R --> S[에러 로그 기록]
    S --> T[결과 데이터 수집]
    Q -->|성공| T
    
    T --> U[결과 분석 및 통계]
    U --> V[HTML 리포트 생성]
    V --> W[마스터 리포트 업데이트]
    W --> X[히스토리 저장]
    X --> Y[브라우저 종료]
    Y --> Z[완료]
```

## 2. 시나리오 실행 상세 플로우

```mermaid
flowchart LR
    A[시나리오 시작] --> B[기본 설정 로드]
    B --> C[환경 변수 설정]
    C --> D[Playwright 초기화]
    D --> E[브라우저 실행]
    E --> F[페이지 로드]
    F --> G[로그인 시도]
    G --> H{로그인 확인}
    H -->|실패| I[재시도 로직]
    I --> G
    H -->|성공| J[테스트 단계 실행]
    
    J --> K[단계 1: 업무코드]
    K --> L[단계 2: 툴체인]
    L --> M[단계 3: 저장소]
    M --> N[단계 4: 사용자]
    N --> O[단계 5: SonarQube]
    O --> P[단계 6: JUnit]
    P --> Q[단계 7: 파이프라인]
    Q --> R[단계 8: 워크플로우]
    
    R --> S{모든 단계 성공?}
    S -->|실패| T[실패 처리]
    S -->|성공| U[성공 처리]
    T --> V[결과 저장]
    U --> V
    V --> W[리포트 생성]
    W --> X[시나리오 완료]
```

## 3. 데이터 처리 플로우

```mermaid
flowchart TD
    A[테스트 실행 완료] --> B[결과 데이터 수집]
    B --> C[성공/실패 통계]
    C --> D[실행 시간 계산]
    D --> E[스크린샷 처리]
    E --> F[로그 파일 생성]
    F --> G[JSON 데이터 저장]
    
    G --> H[global-test-results.json]
    G --> I[scenario-list.json]
    
    H --> J[마스터 리포트 생성]
    I --> J
    J --> K[test_results_master.html]
    
    G --> L[시나리오별 리포트]
    L --> M[custom-report.html]
    L --> N[히스토리 파일]
    
    K --> O[최종 결과물]
    M --> O
    N --> O
```

## 4. 에러 처리 플로우

```mermaid
flowchart TD
    A[테스트 실행 중] --> B{에러 발생?}
    B -->|아니오| C[정상 진행]
    B -->|예| D[에러 타입 분류]
    
    D --> E{네트워크 에러?}
    E -->|예| F[재시도 로직]
    F --> G[최대 3회 재시도]
    G --> H{재시도 성공?}
    H -->|예| C
    H -->|아니오| I[네트워크 에러 기록]
    
    E -->|아니오| J{UI 요소 찾기 실패?}
    J -->|예| K[대체 선택자 시도]
    K --> L[타임아웃 설정]
    L --> M{요소 찾기 성공?}
    M -->|예| C
    M -->|아니오| N[UI 에러 기록]
    
    J -->|아니오| O{데이터 입력 실패?}
    O -->|예| P[입력 필드 재시도]
    P --> Q{입력 성공?}
    Q -->|예| C
    Q -->|아니오| R[데이터 에러 기록]
    
    O -->|아니오| S[기타 에러 기록]
    
    I --> T[에러 로그 저장]
    N --> T
    R --> T
    S --> T
    T --> U[스크린샷 캡처]
    U --> V[테스트 실패 처리]
```

## 5. 사용자 인터페이스 플로우

```mermaid
flowchart TD
    A[GUI 프로그램 시작] --> B[메인 화면]
    B --> C[시나리오 목록 표시]
    C --> D[사용자 시나리오 선택]
    D --> E[설정 파일 확인]
    E --> F{설정 파일 존재?}
    F -->|아니오| G[기본 설정 생성]
    F -->|예| H[설정 로드]
    G --> H
    
    H --> I[실행 버튼 클릭]
    I --> J[테스트 실행 시작]
    J --> K[진행률 표시]
    K --> L[실시간 로그 출력]
    L --> M{테스트 완료?}
    M -->|아니오| K
    M -->|예| N[결과 표시]
    
    N --> O[성공/실패 통계]
    O --> P[리포트 생성 버튼]
    P --> Q[HTML 리포트 생성]
    Q --> R[브라우저에서 리포트 열기]
    R --> S[완료]
```

## 6. 멀티 사용자 처리 플로우

```mermaid
flowchart TD
    A[사용자 목록 로드] --> B[첫 번째 사용자 선택]
    B --> C[사용자 검색]
    C --> D[사용자 클릭 시도]
    D --> E{클릭 성공?}
    E -->|실패| F[대체 선택자 시도]
    F --> G{시도 성공?}
    G -->|실패| H[에러 기록]
    G -->|성공| I[업무코드 부여]
    E -->|성공| I
    
    I --> J[다음 사용자 확인]
    J --> K{더 많은 사용자?}
    K -->|예| L[검색 필드 초기화]
    L --> M[목록으로 돌아가기]
    M --> N[다음 사용자 선택]
    N --> C
    K -->|아니오| O[모든 사용자 처리 완료]
```

## 7. 리포트 생성 플로우

```mermaid
flowchart TD
    A[테스트 결과 수집] --> B[데이터 정규화]
    B --> C[통계 계산]
    C --> D[성공률 산출]
    D --> E[실행 시간 분석]
    E --> F[에러 패턴 분석]
    
    F --> G[마스터 리포트 생성]
    F --> H[시나리오별 리포트 생성]
    
    G --> I[전체 시나리오 현황]
    I --> J[실행 통계 테이블]
    J --> K[히스토리 링크]
    
    H --> L[상세 테스트 결과]
    L --> M[스크린샷 갤러리]
    M --> N[실행 로그]
    N --> O[통계 정보]
    
    K --> P[최종 마스터 리포트]
    O --> Q[시나리오별 상세 리포트]
    
    P --> R[test_results_master.html]
    Q --> S[custom-report.html]
    
    R --> T[리포트 완성]
    S --> T
```

## 8. 설정 관리 플로우

```mermaid
flowchart TD
    A[설정 파일 로드] --> B[기본 설정 확인]
    B --> C[시나리오별 설정 확인]
    C --> D[환경 변수 설정]
    D --> E[프로젝트 정보 로드]
    E --> F[사용자 정보 로드]
    F --> G[저장소 정보 로드]
    
    G --> H{모든 설정 완료?}
    H -->|아니오| I[기본값 적용]
    H -->|예| J[설정 검증]
    
    I --> J
    J --> K{설정 유효성 검사}
    K -->|실패| L[에러 메시지 출력]
    K -->|성공| M[테스트 실행 준비 완료]
    
    L --> N[설정 수정 안내]
    N --> O[사용자 수동 설정]
    O --> A
```

이러한 플로우 차트들은 TROMBONE UI 테스트 자동화 시스템의 전체적인 동작 과정을 시각적으로 보여줍니다. 각 차트는 시스템의 다른 측면을 다루며, 전체적인 이해를 돕습니다.
